# .xogenome Format Specification v0.1

A unified JSON format encoding a creature's visual identity, sonic DNA, personality,
and pixel art data. The atomic unit of the XO_OX Creature Bestiary.

## Governance: B016 — Visual-Sonic Seed Integrity

**Rule**: Once a creature's visual properties have been used to generate sonic DNA
parameters, the sonic DNA is FROZEN. Visual art can be updated (pixel art refinement,
palette tweaks, animation frames) without affecting sonic identity. The creature's
appearance may evolve; its voice does not retroactively change.

**Enforcement**: The `sonic_dna.generated_from_version` field records which visual
version produced the sonic DNA. If `visual.version` advances past this, the sonic
DNA remains at the frozen version unless explicitly regenerated by the user.

## Field Descriptions

### identity
The creature's assignment within the XO_OX ecosystem. `assigned: false` means unassigned
bestiary creature (pool creature, not yet matched to an engine).

### visual
The pixel art data. `version` increments when the art is updated. `palette` and `grids`
are the same format render.py already consumes — backward compatible.

### shape_descriptors
Measured from the analysis-resolution silhouette (128x128). `computed_from_version`
tracks which visual version these were computed from.

### sonic_dna
The Vibe-validated visual-to-sonic mappings. `generated_from_version` enforces B016 —
once frozen, visual updates don't change sonic DNA. `frozen: true` means this is locked.
Set `frozen: false` to allow regeneration on next pipeline run.

### contour
Moore neighborhood traced boundary + extracted waveforms. The waveforms are stored as
2048-sample arrays ready for wavetable export. Phase 2 will add FFT zero-pad resampling.

### personality
Human-authored mythology and character notes. Not auto-generated — filled in from
the creature-list.md descriptions or manually by the user.

### evaluation
Pipeline self-assessment scores. Helps identify which creatures need manual refinement.

## Compatibility

- **render.py**: Reads `visual.grids`, `visual.palette`, `visual.variants` — ignores all
  other fields. Fully backward compatible with existing JSON specs.
- **analyze_reference.py**: Writes the full .xogenome. Existing simple specs still work
  as input to render.py.
- **.xogenome extension**: Files use `.xogenome.json` extension for clarity but are
  standard JSON. Can also be `.json`.

## Architect's Verdict

**APPROVED WITH CONDITIONS:**
1. B016 enforcement via version tracking is acceptable
2. `sonic_dna.frozen` must default to `true` after first generation
3. The `personality` section is human-authored only — no auto-generation of mythology
   (that's `/mythology-keeper`'s domain)
4. `contour.waveforms` are experimental (Phase 2) — clearly labeled as such
