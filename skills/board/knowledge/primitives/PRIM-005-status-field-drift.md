# Status Fields in MEMORY.md Drift Within Days of Completion

Status markers, gallery codes, accent colors, and repo facts in MEMORY.md go stale faster than any other content — often within the same session that produces the change. The problem is systemic: the field was accurate when written, but the truth evolved in a downstream session and nobody updated the pointer.

## Evidence (found 2026-03-14 audit)

- **Gallery code drift**: MEMORY.md calls XOppossum → "BITE" (legacy). CTX-002 and XOppossum/CLAUDE.md both confirm the canonical XOmnibus name is OVERBITE. The correction happened in a prior session but MEMORY was never updated.
- **Gallery code drift (XOverdub)**: XOverdub/CLAUDE.md header says "Gallery Code: DUB" while the Seance verdict in the same file says "XOmnibus Gallery Code: OVERDUB". XOmnibus master spec confirms OVERDUB. The standalone CLAUDE.md was never reconciled with XOmnibus canonical naming.
- **Repo fact drift**: MEMORY.md and xodyssey-engine.md both say "Git repo is ~ (home dir), NOT a dedicated repo." XOdyssey has had its own git repo at `~/Documents/GitHub/XOdyssey/` since Phase 7. The note was accurate during early development and never removed.
- **Preset count drift**: MEMORY.md XOmnibus section says "1,153 total" presets. Actual count is 1,625 .xometa files. The number was correct at the time of writing; multiple subsequent preset generation passes were never reflected.
- **Satellite file count drift**: system-reminder claimed "4 newly designed engines" in the New Engines section; MEMORY.md actually lists 9 engines with satellite files (xorphica, xobbligato, xottoni, xole, xohm, xostinato, xopensky, xoceandeep, xouie).

## Root Cause

Status fields are written at completion time but are never declared stale by the completing session. The next session inherits the stale marker and has no signal that it changed. Unlike architecture docs (which are consulted when building), status fields are skimmed — the eye slides past them.

## Protocol

1. **Every session that changes a status fact must update MEMORY.md in the same commit.** This is a blocking obligation, not a courtesy.
2. **After a Seance verdict, the canonical gallery code from the Seance overrides all prior gallery code references** — update CLAUDE.md, MEMORY.md, and any satellite files in the same session.
3. **Preset counts must never be written as bare integers in prose.** Write "see XOmnibus Presets/ directory" or "generated by [script]", not a hardcoded number.
4. **Repo facts (git location, build command, format) should be verified against the actual filesystem at least once per Board meeting** — not trusted from memory.
